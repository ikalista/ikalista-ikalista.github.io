单词会有许多的形态的变化(morphological variations of words).

  • inflectional (plurals, tenses) 时态, 复数
  • derivational (making verbs nouns etc.) 动词化, 名词化

在最基础的信息检索中, 程序和人不一样, 是无法理解时态和复数的, 程序会将他们分为不同的单词, 从直觉上来说这和我们的语言显然相悖, 所以词干法诞生了.

词干法的两种类型:
  • Dictionary-based: uses lists of related words
  • Algorithmic: uses program to determine related words

词干法会使用算法, 或dictionary based, 将所有的term还原成最原始的状态,

e.g.

does -> do

cups -> cup

在信息检索领域, precision和recall是不能双全的

版权声明：本文引用CSDN博主「飞行codes」的原创文章，遵循CC 4.0 BY-SA版权协议. 原文链接：https://blog.csdn.net/Arctic_Beacon/article/details/115757581

一般来说 recall变高了, precision是会降低的, 所以有了F-measure: trade-off between P & R

Recall = 取到的正例 / ground truth的正例

Percision = 取到的正例 / 取到的所有

F-measure: F-measure: trade-off between P & R

p@i = i处cut off的precision

r@i = i处cut off的recall

AP: Average precision = 求和 p@i / num_rel

AR: Average recall = 求和 r@i

MAP = 多条语句的 AP的和

rank_first_relevant = 第一个relevant的文件在第几个 e.g. 第二个出现, RR = 1/2

RR: Reciprocal rank = 1 / rank_first_relevant

MRR: Mean RR, averaged over queries = 多条语句的RR

Kaggle Digit Recognizer

Posted on 2021-09-23 Edited on 2021-11-16

Kaggle Digit Recognizer

全部流程简介

  • 数据预处理

    获取数据, 从train.csv 和 test.csv里读取

    将train数据打乱并按照8:2的比例分为train set 和 validation set

    将train, valid, test数据分布用torch的Dataset类封装, 从写Dataset的__getitem__() 和__len__()方法

  • 编写模型

    本文用到了一个比较复杂的CNN神经网络

  • 定义神经网络模型

    生成模型对象model=Net()

    定义优化器 optimizer = optim.SGD(model.parameters(), lr=0.01, momentum=0.5)

  • 训练阶段

    循环每个epoch，对于一个epoch：

    • 将模型设置为训练模式 model.train()
    • 将训练数据分为batch_size大小的组，对每一组中所有的样本：
      • 将梯度初始化为0 optimizer.zero_grad()
      • 对这组样本进行训练，并得到输出 output=model(data)
      • 根据预测输出和真实类别计算损失函数 loss=F.nll_loss(output,target)
      • 反向传播，计算梯度 loss.backward()
      • 更新模型参数 optimizer.step()
    • 保存模型（可省略） torch.save(model,”mnist_torch.pkl”)

  • 加载已训练好的模型（可省略）

  • 测试阶段，对于所有的测试样本：

    • 将模型设置为评价模式 model.eval()
    • 根据建好的模型对测试数据进行预测，得到预测结果 output=model(data)
    • 计算损失函数 test_loss+=F.nll_loss(output,target).data
    • 将概率最大的类别当作预测类别，并计算准确率 correct+=pred.eq(target.data).cpu().sum()

上述第四大步的第4小步是当问题是分类问题时才有的

上述中 model.train()和model.eval()的不同在于，model.eval()去除神经网络中的随机性，而model.train()
则保留了神经网络中的随机性。这是因为神经网络中的某些操作（如dropout等），会存在一定的随机性（dropout
会随机让神经网络中的节点失活），所以为了保证预测结果是可复现的，所以在测试阶段要设置为model.eval()。
